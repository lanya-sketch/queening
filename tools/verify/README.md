# 검증 하네스

실제 브라우저로 앱을 구동해 회귀를 확인하는 스크립트 모음.
설치된 Chrome/Edge 를 그대로 쓰므로 브라우저 다운로드가 없다(`playwright-core`).

**모든 스크립트는 개발 서버가 떠 있어야 한다.** 다른 터미널에서 먼저:

```bash
npm run dev
```

## 명령

| 명령 | 하는 일 | 대략 소요 |
| --- | --- | --- |
| `npm run verify` | M1/M2a/M-콘텐츠 회귀 40여 항목 — 착장·초상·모달, 턴 루프, 이벤트, 세이브 연쇄 마이그레이션, 375/812×375/1440 오버플로 | ~1분 |
| `npm run simulate` | 6개 육성 빌드로 각각 9년(40턴) 자동 플레이 → 진실 도달·섭정 관계·국정 영향도 실측 | ~8분 |
| `npm run simulate -- D` | 특정 빌드만 (머리글자 A~F) | ~1.5분 |
| `npm run verify:fallback` | 착장 매니페스트가 깨졌을 때의 폴백 4종 | ~30초 |
| `npm run verify:flags` | 섭정 회유/결렬 flag 배타성 | ~1.5분 |
| `npm run verify:ai` | AI 인프라 — 키 없이도 코어 완전, BYOK 저장/삭제, **브라우저 직접 호출(CORS·헤더)** | ~30초 |
| `npm run verify:providers` | 제공자 어댑터 동등성 — 응답 형식이 달라도 같은 델타·같은 클램핑 (네트워크 불필요) | ~20초 |
| `npm run verify:talk` | 군주 대화 — **대조적인 두 세이브가 다른 페르소나를 만드는지**, 프롬프트에 숫자 없음, 스트리밍·META 비노출, 화면별 델타 허용, 폴백, 잠금 (네트워크 불필요) | ~40초 |
| `npm run gen:outfits` | 플레이스홀더 착장 SVG 8장 재생성 | 즉시 |

## 환경변수

- `QUEENING_URL` — 개발 서버 주소 (기본 `http://localhost:5173/`)
- `QUEENING_CHROME` — Chrome/Edge 실행 파일 경로 (자동 탐색 실패 시)

## 콘텐츠를 추가한 뒤에 볼 것

`npm run simulate` 결과에서 **질적 결론**이 유지되는지가 핵심이다:

- A 빌드 = 깊은 진실 도달, B = 얕은 진실까지(재정 경로), C = 깊은 진실 차단
- D = 회유 성사(동맹), E = 영향도 0까지 떨어져도 회복 가능(함정 없음), F = 상한 근처 도달

**최종 수치가 1~2 흔들리는 것은 정상이다.** 활동 효과에 `variance`(±편차) 난수가 있고
시뮬 플래너가 매 턴 스탯을 읽어 적응하므로, 같은 코드로 두 번 돌려도 값이 미세하게 갈린다.
특히 임계값 기반 정책을 쓰는 D 빌드가 잘 흔들린다. 그러니 **"수치 회귀 0"을 기준으로 삼지 말고**,
질적 결론이 같은지 + 이벤트 조건 임계값이 설계와 일치하는지를 보라.

새 이벤트를 추가했다면 수치는 당연히 그 기여분만큼 달라진다. 그 차이가 새 이벤트로
설명되는지 확인하면 된다.

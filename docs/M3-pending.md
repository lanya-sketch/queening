# M3 인수인계 — 엔딩에서 판정할 것들

여러 라운드에 걸쳐 **flag 만 심어 두고 판정은 미룬** 항목들을 한곳에 모은다.
M3 지시블록은 이 문서에서 거의 그대로 나온다.

공통 원칙: **판정을 앞당기지 않는다.** 국력·국정 영향도·관계는 마지막 계절까지 변하므로,
중간에 가르면 남은 플레이가 결과에 반영되지 않는다. 그래서 지금은 "경로가 열렸다"까지만
기록하고, 갈림은 엔딩 시점에 한다.

이 원칙이 지켜지고 있는지는 검증이 그래프로 확인한다 —
`verify:devices` 가 **아래 예약 flag 를 condition 으로 읽는 이벤트가 0건**임을 대조한다.
읽는 곳이 생기는 순간 판정이 앞당겨진 것이므로 테스트가 잡는다.

---

## 1. ③ 공동왕조 — 동등인가 복속인가

| | |
|---|---|
| 심어진 flag | `union_possible` (M2b-3c-2) |
| 조건 | ③ prince 깊은 관계(70) + 체류 중 |
| M3 가 가를 것 | `union_equal` / `union_subjugated` |

판정 재료(제안): 재정 + 무예 + 국정 영향도의 조합, `legitimacy_sacred`·`military_route_open`
같은 국력 보강 flag. 약한 나라가 결합하면 흡수된다는 것이 이 경로의 위험이다.

## 2. ⑤ 군사노선 — 왕이 이끄는가 군부에 먹히는가

| | |
|---|---|
| 심어진 flag | `military_route_open`, `commander_house_known` (M2b-3c-2) |
| 조건 | ⑤ 호감도 65 + 가문 역사 화제 + 18세 |
| M3 가 가를 것 | `military_king_led` / `military_junta_risk` |

판정 재료(제안): 국정 영향도. **약한 왕에겐 함정, 강한 왕에겐 날개.**
씬에 이미 복선이 있다 — "아홉 대 중 하나는 왕을 갈아치웠습니다."

## 3. 모후의 약 — 발각 배드엔딩 경로

| | |
|---|---|
| 심어진 flag | `queen_poison_path`, `queen_alert_max` (M2b-3c-1) |
| 조건 | 침실 수색 발각 |
| M3 가 할 것 | 실제 '약 시도' 전개 이벤트, 막을 기회, 꼭두각시 배드엔딩 |

★ **지금 이 flag 를 읽는 곳은 어디에도 없다.** 그게 의도다 — 발각의 대가는
"지속 손해"가 아니라 "시간과 확증 루트를 잃는 것"이어야 한다는 설계 결정이었고,
실측으로 재기 가능성(심신 50→100, 영향도 12→36)을 확인했다.
M3 에서 이 경로를 열 때 **재기의 여지를 남기는** 성질을 유지할 것.

## 4. 혈서 확증 — 처분의 근거

| | |
|---|---|
| 심어진 flag | `blood_oath_complete`, `blood_oath_half_*`, `blood_oath_given` / `blood_oath_seized` (M2b-3c-1) |
| M3 가 할 것 | 섭정공·왕대비 처분 분기 |

★ `blood_oath_given`(①이 자발적으로 발설) 과 `blood_oath_seized`(가문 수색으로 강탈) 는
**같은 물건을 정반대 의미로** 얻은 것이다. 확증의 무게는 같아도 그 뒤의 ① 관계는 달라야 한다.

## 5. 두루마리 — 정통성

| | |
|---|---|
| 심어진 flag | `scroll_offered`, `legitimacy_sacred`, `church_support` (M2b-3c-2) |
| 즉시 효과 | 국정 영향도 +18 (유일하게 즉시 보상이 있는 장치) |
| M3 가 할 것 | 친정 엔딩의 정통성 근거, 교회·민심 지지 반영 |

## 6. 섭정 관계 — 동맹 + 높은 의심의 공존

M-콘텐츠 라운드에서 기록해 둔 미결 항목. `regent_alliance` 와 높은 `regentSuspicion` 이
동시에 성립하는 상태의 엔딩 처리가 정해지지 않았다.

## 7. 마지막 턴 처리

`hasReachedEnd` 는 나이 > 20 에서 잠그지만, 그 전환을 만드는 `endTurn` 자체는
이벤트를 처리한다. 그래서 **20세를 넘긴 마지막 턴에도 이벤트가 한 번 발동한다**
(시뮬에서 "즉위 10년 봄 왕 21세 방문"으로 관측됨). 반복 이벤트(`once: false`)가
생기기 전에는 보이지 않던 동작이다. M3 에서 엔딩 진입 처리를 정할 때 함께 결정할 것.

## 라이브 키 테스트 대기 항목

자동 검증으로는 확정할 수 없고 **실제 모델 응답이 필요한** 판단들.
`QUEENING_LIVE_KEY=sk-ant-... npm run verify:chars` 로 실제 대화를 돌린 뒤 정한다.

1. **다섯 캐릭터의 목소리 분화** (M2b-3b-2)
   시스템 프롬프트가 서로 다르다는 것은 실측했다. 모델이 그 시트대로 실제로 다르게
   말하는지는 키 없이 확인 불가.

★ 두 항목 모두 **키를 넣으면 바로 돌아가는 스크립트가 준비돼 있다.** 막연한 TODO 가 아니다.
   각 스크립트의 배선은 네트워크를 가로채 검증했고, 남은 미지수는 모델의 실제 행동뿐이다.

2. **③ 깊은 관계 도달 속도** (M2b-3c-2)
   → `QUEENING_LIVE_KEY=sk-ant-... npm run verify:live-affection`
   회당 호감도 상승을 실측해 70 까지 몇 회가 필요한지 계산하고,
   세션 호출 상한(60)을 넘으면 조정 후보를 제시한다.
   상한 계산으로는 도달 가능하다 — 호감도 5→70 = 65점, 회당 상한 +3 → 최소 22회 응답,
   접근량은 2.5회 방문 × 체류 2턴 = 5세션이고 **한 턴 안의 대화 횟수에는 상한이 없다.**
   그러나 이건 상한 계산이지 실측이 아니다. 시뮬 플래너는 대화를 시뮬레이션하지 않는다.
   실제 회당 델타 기대값이 낮으면 세션 호출 소프트 상한(60)에 먼저 걸린다.
   → 실측 후 너무 느리면 **델타 상한(MAX_AFFECTION 3) 또는 체류 기간**을 조정.
   체류 연장은 병목이 아닐 가능성이 높다(턴당 메시지 무제한).
   화제 해금의 결정론적 보상(+8)이 이 부담을 얼마나 덜어 주는지도 함께 볼 것.

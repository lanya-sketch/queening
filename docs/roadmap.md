# 완성까지 남은 작업 전체 지도

이 문서가 로드맵의 단일 출처다. 예전에 여러 문서(`M2b-0` 끝, `M3-pending.md`)에
흩어져 있던 미래 작업을 전부 여기로 모았다. 새 방향은 여기에 먼저 적는다.

원칙은 그대로다: **각 라운드는 설계 제안 → 확인 → 구현 → 실측 검증 → 정직한 보고.**
"검증 안 한 것을 했다고 하지 않는다"가 이 프로젝트를 여기까지 끌고 온 규칙이다.

## 진행 순서 (확정)

### 1. 하드 배타성 + 숙청 대안 — ✅ **완료** (실측 로그 있음)

> 결정적 씬(호감도 70 → 확정/거절, `romance_settled` 마스터 게이트) + 청산 4종
> (①처형/②희생양/④고립/⑤반역, ③무관) + 호감도 구간별 후일담(⑤ 3구간, ①②④ 높은구간).
> M3-1 romanceOf 확정 flag 기반 단순화, 엔딩 @purge anchor·숙청 수식(청산/고독한 옥좌/피 묻은 손).
> 검증: verify:hard-exclusive, 판정 완전성 1만 세이브, **hardexclusive ablation 10빌드 미스터리 무손상**.

로맨스를 느슨한 배타성 → **하드 배타성**으로. 코어 메커니즘 변경이라 최우선.

- **결정적 씬**: 각 캐릭터 관계 확정 순간. 받아들이면 그 루트로 확정되고 나머지 로맨스는 닫힌다.
- **3단 구조**: 탐색기(자유, 여러 명 동시 가능) → 결정적 씬(확정) → 확정 후(한 명).
- **시간이 자연적 배타 압력**: 늦게 열리는 고유장치는 시간 부족으로 다 못 간다.
- **고유장치는 로맨스 없이도 유지**되되, 그때는 "도구/숙청"으로 쓰인다.
- ★ **다섯 숙청 대안** — 로맨스 안 한 캐릭터를 그 정체성으로 역이용. 전부 "선택":
  - ① 역적의 핏줄로 처형 / ② 급진 계승자로 희생양 / ④ 작위로 고립 / ⑤ 반역 혐의로 청산
  - ③ 은 외부인이라 무관 — 추가 장치 없음
- ★ **관용도 선택이되 대가가 있다**: 살려둠 = 위험 감수(① 훗날 불씨, ⑤ 갈아치울 힘 잔존 등).
- 새 flag: `heir_executed`/`heir_spared`, `loyalist_scapegoat`, `hero_isolated`, `commander_purged` 등.
- **M3 반영**: 각 처분이 엔딩 삽입으로(잔혹/관용의 색). **M3-1 판정은 로맨스 최대 하나로 단순화.**

### 1-b. 측실 갈래 + ③ 정복 병합 — ✅ **완료** (실측 로그 있음)

> 청산 3갈래(죽임/측실/관용): ⑤ 보검 압수 + 3구간 후일담, ①②④ flag + 실제 삽입.
> '소유의 옥좌' 수식. ③ 정복 능동 이벤트(17세+/국력 압도/로맨스 무관), 실행 시 호감도 급락 +
> `prince_conquered` 가 union 판정 가로챔. 네 갈래(공동왕조/사랑을 삼킴/무감정 정복/무관).
> 검증: verify:concubine, 완전성 1만 세이브, hardexclusive ablation(측실·정복 포함) 무손상,
> K빌드로 측실 후일담 실제 렌더 확인(정의 검증 ≠ 렌더 검증).

- 청산 선택(현재 죽임/관용)에 **"측실" 세 번째 갈래** 추가. "애매하게 선택받은 절망" —
  사랑도 자유도 죽음도 아닌, 곁에 묶어둔 소유. **호감도 높을수록 절망이 깊다.**
- ★ 원칙:
  - **강압을 미화하지 않는다** — 당한 자의 절망이 핵심이다.
  - **미성년 배제** — 청년기·성인만.
  - **엔딩 서술에만 함축적으로** — 자세히 쓰지 않음(15세+·성인 커뮤니티라 해석 여지는 남김).
  - **게임플레이 시스템화 안 함** — flag + 엔딩 삽입만.
- 갈래 통합: **죽임 / 측실(감정 + 정치적 소유 겸함) / 관용**의 3갈래.
- **③ 정복 병합**: 로맨스 없이 국력이 압도하면 ③의 나라를 정복·속국화(제국 참칭 실현).
  `배드:제국복속`의 **거울상**(우리가 복속시키는 쪽). 개인이 아니라 국가 정복.

### 2. D 시스템 — 게임을 게임답게

- **D-1 ✅ 완료** — 타이틀 화면(빈 옥좌 + 새 게임/이어하기/설정·갤러리 자리) +
  서사형 온보딩(왕당파 노귀족이 규칙·세계·정체성을 대사로, 툴팁 3개 AP·게이지·턴종료,
  스킵 가능). 앱 레벨 라우팅(`appStore`, 엔진 turn 과 분리). 하네스 `enterGame` 헬퍼.
  검증: verify:title. (세이브는 단일 슬롯 유지 — 다중 슬롯은 별도 라운드.)
- **D-2 남음**: 엔딩 갤러리(수집) + 결산 차등 표시(스탯 등 엔딩별로).
- **D-3 남음**: 옵션(텍스트 속도·스킵), 진행 가시성(초상 배경 변화로 루트 암시).

### 2-b. 조기 배드엔딩 — 20세 전 파국 (D 시스템 이후쯤)

현재 배드엔딩은 전부 20세 판정. 게임 **중간에 파국으로 끝나는 조기 배드**를 추가해
생존 긴장을 준다.

- ★ 원칙: **"가혹하되 좌절 아닌"** — 예고 없이 걸리지 않는다. 위험 신호(심신 낮음·의심 높음)가
  **누적·방치될 때만**, 그리고 **회피 기회 1회**.
- 후보(미확정): 심신 장기 파탄 / 의심 최대 + 무방비(`queen_poison_path` 확장) /
  실권 완전 상실 / 정치 현안 연속 파국 / ⑤ 무신정변 선제 · ① 아버지 반격.
- 시점: **코어·밸런스 안정 후**(D 시스템 이후쯤). 씨앗은 이미 있다 — `queen_poison_path`(3c-1).

### 3. 콘텐츠 채우기

- 캐릭터 **2·3층 호감도 중간 구간** 서술(손으로, 급하지 않음).
- **군주 착장 배선**: 배경 투명화 완료된 남/녀 나이대별 에셋을 매니페스트에.
  - ★ 매니페스트에 **나이대 × 성별 축 확장**, 런타임 JSON 교체 성질 유지,
    축 없는 기존 매니페스트 **하위호환**(`verify:fallback` 케이스 추가).
- **5인 초상**: ① 3벌(13/16/청년) · ② 2벌(소녀/청년) · ③④⑤ 각 1벌. 착장 방향:
  - ① 귀족 정장(절제) / ② 단정한 영애복(수수) / ③ 제국풍 무장(오만) /
    ④ 낡은 군장(포상 거부) / ⑤ 친위대 군복·갑주

### 4. 논의 후 결정

- **시작 기질 선택** — 트레이드오프 프리셋, 새 지표 없이. 설계 초안은 [M2b-0](M2b-0) 끝에 있음.
- **회차 계승** — 고민 중.

### 5. 텍스트 중앙화 → 동양판 ⟵ **최후**

서양판 텍스트가 **전부 완성된 뒤**. 컴포넌트 하드코딩 문자열을 `data/` 로 모으고,
그 위에 동양판 텍스트를 교체한다. 엔딩은 이미 `data/endings/` 에 있으나
다른 화면들(결산 문구 등)은 점검이 필요하다.

### 6. 배포 패키징 ⟵ **완성 후 맨 마지막**

- 현재 스택(Vite + React + TS, **순수 프론트엔드, 서버 없음, localStorage**)이
  웹/EXE/APK 를 전부 열어 둔다. **지금 바꿀 것 없음.**
- 웹: `vite build` → itch.io 등(제일 쉬움).
  EXE: Electron 또는 Tauri. APK: Capacitor 또는 Tauri 2.0.
- ★ 배포 시 확인: **BYOK CORS 가 각 웹뷰에서 되는지**, 세이브 localStorage 유지 vs 파일 기반.
- 시점: 게임·콘텐츠 **완성 후 맨 마지막**.

## 라이브 키 대기 (사용자가 병행 확인)

자동 검증으로는 확정 불가, **실제 모델 응답이 필요.** 스크립트는 준비돼 있고
배선은 네트워크 가로채기로 검증됨 — 남은 미지수는 모델의 실제 행동뿐이다.

- **다섯 캐릭터 실제 목소리 분화** → `QUEENING_LIVE_KEY=sk-ant-... npm run verify:chars`
- **③ 깊은 관계 도달 속도** → `QUEENING_LIVE_KEY=sk-ant-... npm run verify:live-affection`
- 실제 호감도 상승 속도로 밸런스 조정 여부(느리면 `MAX_AFFECTION 3`·체류·화제 보상 조정).

상세는 [M3-pending.md](M3-pending.md) 끝의 "라이브 키 테스트 대기 항목".
